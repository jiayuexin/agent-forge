# 3. Agent 生成引擎

> 从岗位描述到可运行 Agent 代码的自动生成流程。

> 工具名命名规范: kebab-case (如 query-order, create-refund)。生成的 Agent 包名: agent-<role> (如 agent-customer-service)。

## 3.1 生成流程

```
用户输入描述
     │
     ▼
┌─────────────┐
│ 1. 描述解析  │  提取关键信息：岗位名称、核心能力、适用场景
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 2. 模板匹配  │  根据描述匹配最合适的预置模板（可跳过使用通用模板）
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 3. Prompt   │  基于岗位描述 + 模板 → 生成系统提示词
│    生成     │  包含：角色设定、行为规范、输出格式、限制条件
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 4. 工具推荐  │  根据岗位需求推荐适用的工具集
│             │  如：客服 → 订单查询、退款处理、通知发送
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 5. 代码生成  │  使用 EJS 模板引擎渲染 Agent 源码
│             │  生成：index.ts, prompts.ts, tools.ts, types.ts
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 6. 项目配置  │  生成 package.json, tsconfig.json, .eslintrc
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 7. 文档生成  │  生成 README.md, API 文档, 使用示例
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 8. 验证检查  │  TypeScript 编译检查, 依赖完整性检查
└──────┬──────┘
       │
       ▼
   ✅ 生成完成
```

## 3.2 Prompt 生成策略

生成的系统提示词包含以下模块：

```markdown
# {角色名称}

## 身份定义
你是一个{岗位描述}...

## 核心能力
1. {能力1}
2. {能力2}
...

## 行为规范
- {规范1}
- {规范2}
...

## 工具使用
你可以使用以下工具：
{工具列表及说明}

## 输出格式
{结构化输出定义}

## 限制条件
- {限制1}
- {限制2}
...
```

## 3.3 预置模板库（初始版本）

| 模板 ID | 名称 | 适用场景 | 内置工具 |
|---|---|---|---|
| `customer-service` | 客服专员 | 用户咨询、投诉处理 | 订单查询、退款、通知 |
| `sales-assistant` | 销售顾问 | 产品推荐、报价生成 | 产品搜索、报价生成 |
| `code-reviewer` | 代码审查员 | 代码质量审查 | 文件读取、Lint 执行 |
| `content-writer` | 内容创作者 | 文案撰写、翻译 | 无（纯文本） |
| `data-analyst` | 数据分析师 | 数据查询、报表分析 | 数据库查询、图表生成 |
| `general` | 通用 Agent | 未匹配其他模板时 | 无 |
