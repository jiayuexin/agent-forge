> 📌 本文件包含两部分：**#4 集成方案**（§4.1–§4.6）和 **#5 Agent 编排协作**（§5.1–§5.8）。

## 4.2 方式 1：npm 包集成（Node.js 项目）

```typescript
// 1. 安装
// npm install ./agents/agent-customer-service

// 2. 使用
import { CustomerServiceAgent } from 'agent-customer-service';

const agent = new CustomerServiceAgent();
await agent.init({
  model: {
    provider: 'openai',
    modelName: 'gpt-4o',
    apiKey: process.env.OPENAI_API_KEY,
  },
});

const result = await agent.execute({
  type: 'chat',
  input: { message: '我要退货' },
});

console.log(result.output.content);
await agent.destroy();
```

## 4.4 方式 3：SDK 集成（多 Agent 协作）

```typescript
import { AgentFramework } from '@agentforge/sdk';

const framework = new AgentFramework({
  agentDir: './agents',
  modelRegistry: {
    endpoints: [{
      id: 'openai',
      baseUrl: 'https://api.openai.com/v1',

  type: 'chat',
  input: { message: '用户想升级套餐' },

const pipelineResult = await framework.pipeline()
  .add('customer-service', { task: '分析用户需求' })
  .add('sales-assistant', { task: '推荐升级方案' })
  .run();
```

# 5. Agent 编排协作

> 核心理念：每个 Agent 是独立 npm 包，互相不认识。多 Agent 协作在宿主项目中通过 `@agentforge/sdk` 完成。

## 5.1 架构

```
┌─────────────────────────────────────────────────────────────┐
│                     宿主项目 (你的业务系统)                    │
│                                                              │
│   import { AgentFramework } from '@agentforge/sdk';         │
│                                                              │
│   ┌────────────────────────────────────────────────────┐    │
│   │          @agentforge/sdk（编排中心）                 │    │
│   │   Pipeline / EventBus / 直接调用                     │    │
│   └─────┬──────────────┬──────────────┬────────────────┘    │
│         │              │              │                      │
│   ┌─────▼────┐  ┌──────▼─────┐  ┌───▼───────┐            │
│   │ 客服Agent │  │ 销售Agent   │  │ 数据Agent  │           │
│   │ (npm包)   │  │ (npm包)     │  │ (npm包)    │           │
│   └──────────┘  └────────────┘  └────────────┘            │
└─────────────────────────────────────────────────────────────┘
```

## 5.2 编排模式一：Pipeline 流水线

### 顺序执行

```typescript
const result = await framework
  .pipeline('处理用户投诉')
  .add('service', {
    task: '分析用户投诉内容，提取关键信息',
    input: { message: '你们的商品质量太差了，我要退款！' },
  })
  .add('sales', {
    task: '根据分析结果，生成退款或补偿方案',
  })
  .add('service', {
    task: '整合方案，生成对用户的最终回复话术',
  })
  .run();
```

### 数据流转

```
Agent A (客服-意图分析)
  │ output: { content: "...", structured: { intent: "refund", product: "运动鞋" } }
  ▼ 自动传递
Agent B (销售-方案推荐)
  │ input: ← 上一步的完整 output
  │ output: { structured: { refundAmount: 299, coupon: 50 } }
  ▼ 自动传递
Agent C (客服-话术生成)
  │ input: ← 前两步的 output 合并
  ▼
最终结果 → 宿主项目
```

### 精确控制数据传递

```typescript
.add('sales', {
  task: '生成方案',
  transform: (prevOutput) => ({
    intent: prevOutput.structured.intent,
    product: prevOutput.structured.product,
  }),
})
```

### 并行执行

```typescript
const report = await framework
  .pipeline('每日运营报告')
  .parallel([
    { agent: 'data', task: '生成昨日销售数据报表' },
    { agent: 'service', task: '总结昨日客服工单统计' },
    { agent: 'sales', task: '分析昨日转化漏斗数据' },
  ])
  .add('data', {
    task: '整合以上三份报告，生成一份完整的每日运营报告',
  })
  .run();
```

### 条件分支

```typescript
const result = await framework
  .pipeline('智能客服路由')
  .add('service', { task: '判断用户意图' })
  .branch((prevOutput) => {
    const { intent } = prevOutput.structured;
    switch (intent) {
      case 'refund':    return { agent: 'sales', task: '处理退款申请' };
      case 'complaint': return { agent: 'manager', task: '升级至主管处理' };
      case 'inquiry':   return { agent: 'service', task: '解答常见问题' };
      default:          return { agent: 'service', task: '生成友好回复' };
    }
  })
  .run();
```

## 5.3 Pipeline 回退与跳转

> 串行流水线中，下游 Agent 发现上游输出有问题时，需要能退回重做或跳转到指定步骤。这是实际生产场景中的关键能力。

### 设计原则

- **可控的回退**：不允许无限循环，必须设最大回退次数
- **上下文传递**：回退时携带当前 Agent 的反馈信息，让上游知道"哪里出了问题"
- **快照机制**：每个步骤执行完保存快照，回退时可以精确恢复到任意步骤的状态
- **可视化**：回退操作在 Dashboard 流水线面板中可见

### 回退信号（Agent 主动回退）

下游 Agent 在 `execute()` 中返回特殊的回退指令，SDK 自动处理跳转：

```typescript
const result = await framework
  .pipeline('合同审批流程')
  .add('draft', {
    task: '根据用户需求起草合同',
  })
  .add('review', {
    task: '审查合同条款，检查合规性和风险',
    // review Agent 如果发现草案有问题，可以返回回退信号
  })
  .add('sign', {
    task: '生成签署文件',
  })
  .run();

// review Agent 内部逻辑示例：
async execute(task: AgentTask): Promise<AgentResult> {
  const issues = this.analyzeContract(task.input);

  if (issues.critical) {
    // 发现严重问题，退回起草步骤，附带修改建议
    return {
      output: { content: '合同存在严重问题', structured: { issues } },
      control: {
        action: 'back',
        targetStep: 'draft',              // 退回到哪一步
        message: '以下条款不合规，请修改：...', // 传给 draft Agent 的反馈
        context: { issues, suggestions }, // 附带的上下文数据
      },
    };
  }

  // 没问题，继续往下
  return { output: { content: '审查通过' } };
}
```

### 外部拦截回退（宿主项目控制）

宿主项目可以在流水线步骤间插入拦截器，根据输出结果决定是否回退：

```typescript
const result = await framework
  .pipeline('数据处理流水线')
  .add('extract', { task: '从文档中提取结构化数据' })
  .add('validate', { task: '验证数据完整性和格式' })
  .add('transform', { task: '数据转换和标准化' })
  .add('load', { task: '写入目标数据库' })
  .intercept('validate', (output, context) => {
    // 检查验证结果
    if (output.structured.errorCount > 5) {
      // 错误太多，退回提取步骤重做
      return {
        action: 'back',
        targetStep: 'extract',
        message: `发现 ${output.structured.errorCount} 个错误，请重新提取`,
        maxRetries: 2,  // 最多重试 2 次
      };
    }
    if (output.structured.warningCount > 10) {
      // 警告较多但可继续，跳过转换直接加载
      return {
        action: 'jump',
        targetStep: 'load',
        message: '跳过转换步骤，直接写入',
      };
    }
    return { action: 'continue' }; // 正常继续
  })
  .run();
```

> 以下类型定义参见 [01-核心设计.md §1.8](01-核心设计.md#18-pipeline-编排类型)：`PipelineControlSignal`、`StepSnapshot`、`BacktrackEvent`、`PipelineResult`

### 模型注册表与路由

Pipeline 编排的核心能力：**Agent 只写模型名，编排器自动路由到正确的端点。**

#### 模型注册表

框架初始化时注册所有模型端点（参见 [01-核心设计.md](01-核心设计.md) 1.4 节完整类型定义）：

```typescript
const framework = new AgentFramework({
  modelRegistry: {
    endpoints: [
      {
        id: 'openai-official',
        baseUrl: 'https://api.openai.com/v1',
        provider: 'openai',
        apiKey: process.env.OPENAI_API_KEY,
        models: ['gpt-4o', 'gpt-4o-mini', 'o1'],
      },
      {
        id: 'openai-proxy',
        baseUrl: 'https://my-proxy.example.com/v1',
        provider: 'openai',
        apiKey: process.env.PROXY_API_KEY,
        models: ['gpt-4o'],
      },
      {
        id: 'ollama-local',
        baseUrl: 'http://localhost:11434',
        provider: 'ollama',
        models: ['qwen2.5:14b', 'deepseek-r1:14b'],
      },
    ],
    defaultEndpoint: 'openai-official',
    defaultModel: 'gpt-4o',
  },
});
```

#### Agent 引用模型

Pipeline 中 Agent 只需写模型名或 `ModelRef`：

```typescript
await framework
  .pipeline('智能客服')
  .config({
    defaultModel: 'gpt-4o',              // 流水线级默认模型
  })
  .add('intent', {
    task: '判断用户意图',
    model: 'gpt-4o-mini',                // 简写：自动匹配 openai-official 端点
  })
  .add('analyze', {
    task: '深度分析问题',
    model: {                             // 指定端点
      model: 'gpt-4o',
      endpoint: 'openai-proxy',          // 走代理端点
    },
  })
  .add('draft', {
    task: '起草回复',
    model: 'deepseek-r1:14b',            // 自动匹配 ollama-local 端点
  })
  // intent 和 draft 没指定 model → 使用 defaultModel 'gpt-4o'
  .run();
```

#### 路由解析规则

```
1. Agent 指定了 model + endpoint → 直接用该端点
2. Agent 只指定了 model 字符串 → 注册表中查找第一个包含该模型的端点
3. Agent 没指定 model → 用 Pipeline 级 defaultModel
4. 匹配不到 → 报错 ModelNotFoundError
```

#### Agent 单独运行

Agent 不在 Pipeline 中时，直接用 `ModelConfig` 配置一个端点即可：

```typescript
// 单独跑，自己管自己的模型
await agent.init({
  model: {
    provider: 'ollama',
    modelName: 'qwen2.5:14b',
    baseUrl: 'http://localhost:11434',
  },
  systemPrompt: '...',
});
```

### 全局配置

```typescript
const result = await framework
  .pipeline('审批流程')
  .add('step1', { task: '...' })
  .add('step2', { task: '...' })
  .add('step3', { task: '...' })
  .config({
    defaultModel: 'gpt-4o',       // Pipeline 级默认模型（可选，覆盖框架全局）
    maxBacktracks: 5,             // 全局最大回退次数，防止无限循环
    defaultMaxRetry: 2,           // 每步默认最大重试次数
    snapshotEnabled: true,        // 是否保存快照（默认 true）
    backtrackLogging: 'verbose',  // 回退日志级别：silent | summary | verbose
    onBacktrack: (event) => {  // 回退回调，可用于通知/告警
      console.warn(`回退: ${event.fromStep} → ${event.toStep}，原因: ${event.reason}`);
    },
  })
  .run();
```

### 分叉能力（高级）

从某个步骤分叉出并行分支，对比不同处理路径的结果：

```typescript
const result = await framework
  .pipeline('方案评审')
  .add('analyze', { task: '分析用户需求' })
  .fork('analyze', [
    { branchName: 'plan-a', steps: [
      { agent: 'architect', task: '生成方案 A（微服务架构）' },
      { agent: 'reviewer', task: '评审方案 A' },
    ]},
    { branchName: 'plan-b', steps: [
      { agent: 'architect', task: '生成方案 B（单体架构）' },
      { agent: 'reviewer', task: '评审方案 B' },
    ]},
  ])
  .add('decider', { task: '对比两个分支的评审结果，选择最优方案' })
  .run();

// decider 会收到合并后的输入：
// {
//   planA: { output: '...', reviewResult: '...' },
//   planB: { output: '...', reviewResult: '...' },
// }
```

## 5.4 编排模式二：事件总线（松耦合异步）

```typescript
// 订阅事件
framework.on('order:created', async (orderData) => {
  await framework.run('service', {
    task: '发送订单确认通知',
    input: { orderId: orderData.id, userId: orderData.userId },
  });
});

framework.on('order:created', async (orderData) => {
  await framework.run('sales', {
    task: '根据购买商品推荐关联商品',
    input: { items: orderData.items },
  });
});

// 发布事件
app.post('/api/orders', async (req, res) => {
  const order = await createOrder(req.body);
  framework.emit('order:created', order);
  res.json(order);
});
```

**事件总线 API：**

| 方法 | 说明 |
|---|---|
| `on(event, handler)` | 订阅事件（可多个订阅者） |
| `once(event, handler)` | 一次性订阅 |
| `off(event, handler)` | 取消订阅 |
| `emit(event, data)` | 发布事件 |

## 5.6 编排模式对比

| 模式 | 耦合度 | 适用场景 | 配置复杂度 |
|---|---|---|---|
| Pipeline 流水线 | 中 | 固定流程、多步骤协作、数据需要逐步传递 | 低 |
| 事件总线 | 低 | 异步通知、一对多广播、触发式响应 | 低 |
| 直接调用 | 高 | 简单的 A 调 B、一次性场景 | 最低 |

## 5.7 SDK 核心 API

```typescript
interface AgentFramework {
  register(name: string, AgentClass: new () => IAgent): this;
  get(name: string): IAgent;
  init(config: AgentConfig): Promise<void>;
  run(name: string, task: AgentTask): Promise<AgentResult>;
  pipeline(name?: string): Pipeline;
  on(event: string, handler: Function): this;
  once(event: string, handler: Function): this;
  off(event: string, handler: Function): this;
  emit(event: string, data: any): void;
}

// PipelineResult 定义见 01-核心设计.md §1.8
interface Pipeline {
  add(agentName: string, step: PipelineStep): this;
  parallel(steps: PipelineStep[]): this;
  branch(condition: (prevOutput: AgentResult) => PipelineStep | Pipeline): this;
  transform(fn: (prevOutput: AgentResult) => any): this;
  intercept(stepName: string, handler: (output: AgentResult, context: any) => PipelineControlSignal): this;
  fork(stepName: string, branches: ForkBranch[]): this;
  config(options: PipelineConfig): this;
  run(): Promise<PipelineResult>;
}

interface PipelineStep {
  task: string;
  input?: Record<string, any>;
  timeout?: number;
  model?: string | ModelRef;       // 模型名（自动路由）或 { model, endpoint }
  transform?: (prevOutput: AgentResult) => any;
  intercept?: (output: AgentResult, context: any) => PipelineControlSignal;
}
```

## 5.8 实际业务集成示例

**电商智能客服系统：**

```typescript
import { AgentFramework } from '@agentforge/sdk';
import { CustomerServiceAgent } from 'agent-customer-service';
import { SalesAssistantAgent } from 'agent-sales-assistant';

const app = express();
const framework = new AgentFramework();

framework.register('service', CustomerServiceAgent);
framework.register('sales', SalesAssistantAgent);
await framework.init();

app.post('/api/chat', async (req, res) => {
  const result = await framework
    .pipeline('客服对话')
    .add('service', {
      task: '回复用户消息',
      input: { message: req.body.message, userId: req.body.userId },
    })
    .branch((prev) => {
      if (prev.output.structured?.opportunity) {
        return {
          agent: 'sales',
          task: '基于上下文推荐商品',
        };
      }
      return null;
    })
    .run();

  res.json({ reply: result.output.content });
});
```
