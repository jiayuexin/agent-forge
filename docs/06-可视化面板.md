# 6. 可视化面板设计

> Dashboard 技术选型、页面设计、Agent 调试台、分离部署监控。

## 6.1 技术选型

| 层面 | 选型 | 原因 |
|---|---|---|
| 框架 | React 18 + TypeScript | 生态成熟，类型安全 |
| 构建 | Vite | 快速，轻量 |
| UI 库 | Ant Design 5 + TailwindCSS 4（样式辅助） | 企业级组件丰富 + 原子化样式 |
| 图表 | ECharts | 中文文档友好 |
| 状态管理 | Zustand | 轻量，简单 |
| 通信 | Axios + 原生 WebSocket | 够用即可 |

## 6.2 页面结构

```
/ (首页/概览)
├── /agents              # Agent 列表
│   ├── /agents/:id      # Agent 详情/配置
│   └── /agents/create   # 创建 Agent
├── /playground          # Agent 调试台
├── /pipeline            # 流程编排 (P1)
└── /monitor             # 监控面板 (P1)
```

## 6.3 核心页面说明

**首页/概览 `/`**

- Agent 数量统计、运行状态概览
- 近期活动时间线
- 快速创建入口

**Agent 列表 `/agents`**

- 卡片式展示所有 Agent
- 显示：名称、状态、岗位、最近调用时间
- 操作：配置、测试、启停、删除

**创建 Agent `/agents/create`**

- 表单输入：名称、描述、模型选择、工具勾选
- 实时预览生成的 Prompt
- 创建成功后跳转到详情页

## 6.4 Agent 调试台 `/playground`

生成 Agent 后的核心验证环境，支持交互式对话测试和深度调试。

**页面布局（三栏）：**

```
┌─────────────────────────────────────────────────────────────────────┐
│  Agent 调试台                                        [⚙ 配置] [📤 导出]  │
├──────────────┬──────────────────────────┬───────────────────────────┤
│   会话面板    │       对话区域            │      调试面板              │
│              │                          │                           │
│  📋 对话历史   │  👤 用户: 帮我审查这段...    │  📊 本次调用统计            │
│  ├─ 会话 1    │                          │  ├─ 响应耗时: 2.3s        │
│  ├─ 会话 2    │  🤖 Agent: 我发现以下...  │  ├─ Token: 1,245         │
│  └─ 会话 3    │                          │  ├─ 模型: gpt-4           │
│              │                          │  └─ 成本: $0.008         │
│  ⚙ 调试配置   │                          │                           │
│  ├─ 模型选择   │  ┌──────────────────┐   │  🔍 调用链路               │
│  ├─ 温度      │  │  输入消息...       │   │  ├─ System Prompt 构建 ✅  │
│  ├─ MaxTokens │  │                  │   │  ├─ LLM 调用 ✅           │
│  ├─ 工具开关   │  └──────────────────┘   │  ├─ 工具调用: query-order    │
│  └─ 🔌 插件市场 │  [发送] [清空] [重试]      │  ├─ 二次 LLM 调用 ✅      │
│              │                          │  └─ 结构化输出解析 ✅      │
└──────────────┴──────────────────────────┴───────────────────────────┘
```

**核心功能：**

| 功能 | 说明 |
|---|---|
| Agent 选择 | 顶部下拉选择要测试的 Agent，支持按名称/标签搜索 |
| 实时对话 | 流式输出（SSE），支持 Markdown 渲染、代码高亮、LaTeX 数学公式 |
| 调试配置 | 可实时切换模型（OpenAI / Anthropic / Ollama）、调整温度、MaxTokens、开关特定工具，无需重启 Agent |
| 工具插拔 | 调试时动态挂载/卸载工具插件，测试 Agent 在不同工具组合下的表现 |
| 临时 Mock 工具 | 不接入真实后端，用自定义 Mock 数据模拟工具返回，测试 Agent 的工具调用逻辑 |
| 调用链路追踪 | 展示每次对话的完整调用链：System Prompt 构建 → LLM API 调用 → 工具调用（如有） → 二次 LLM → 结构化输出解析 |
| 工具调用详情 | 点击调用链路中的工具节点，查看入参、出参、耗时 |
| Token 统计 | 实时显示 Prompt Tokens / Completion Tokens / Total Tokens，估算成本 |
| 会话管理 | 保存多组对话会话，可切换、清空、导出为 JSON |
| 变量注入 | 支持设置上下文变量（如 `{orderId}`），对话中可引用，模拟真实业务场景 |
| 对比测试 | 同一输入，同时发给不同模型或不同配置的 Agent，左右对比输出效果 |
| 导出报告 | 将对话记录 + 调用链路 + 统计数据导出为 Markdown / JSON 报告，便于分享和存档 |

**调试台后端 API：**

> 调试台 API 端点定义参见 [05-CLI与API.md §5.3](05-CLI与API.md#53-http-api-接口)

> 调试台相关类型定义（`DebugConfig`、`InjectedTool`、`MockToolConfig`、`CallTrace`）见 [01-核心设计.md](01-核心设计.md) 第 1.9 节。

# 7. 面板与 Agent 分离部署监控

> 当面板和 Agent 不在同一进程甚至不同机器时的监控方案。

## 7.1 架构总览

```
┌──────────────┐          ┌──────────────────┐          ┌──────────────┐
│  Web 管理面板  │          │  Agent 节点 1      │          │  Agent 节点 2  │
│  :8080        │  HTTP    │                  │  HTTP    │              │
│              │◄────────►│  Agent A :3001   │◄────────►│  Agent D     │
│  React 前端   │  WebSocket│  Agent B :3002   │  WebSocket│  Agent E     │
│              │          │  Agent C :3003   │          │              │
└──────┬───────┘          └──────────────────┘          └──────────────┘
       │
       ▼
┌──────────────┐
│  Dashboard   │
│  后端服务     │  ← 独立 Node.js 服务，负责注册发现 + 指标聚合 + 事件转发
│  :8080       │
└──────────────┘
```

**通信链路：**

```
Agent ◄──── 心跳/指标采集 ────► Dashboard 后端 ◄──── REST/WS ────► 面板前端
Agent ──── 注册/注销 ─────────► Dashboard 后端
Agent ──── 实时事件推送 ──────► Dashboard 后端 ──── WebSocket ────► 面板前端
```

## 7.2 Agent 端：启动时注册 + 心跳上报

```typescript
import { serve } from '@agentforge/core';

const server = await serve({
  agent: myAgent,
  port: 3001,
  registry: {
    dashboardUrl: 'http://dashboard-host:8080',
    name: 'agent-customer-service',
    tags: ['客服', '售后'],
  },
  events: {
    dashboardUrl: 'ws://dashboard-host:8080/ws/events',
  },
});

// 启动后自动执行：
// 1. 向 Dashboard POST /api/agents/register → 注册
// 2. 每 30 秒 POST /api/agents/heartbeat → 心跳
// 3. 通过 WebSocket 推送 execute/error/tool:call 等事件
// 4. 关闭时 DELETE /api/agents/unregister → 注销
```

## 7.4 Dashboard 后端：注册发现 + 指标聚合

```typescript
const agentRegistry = new Map<string, AgentNode>();

// Agent 注册
app.post('/api/agents/register', (req, res) => {
  const { name, url, tags, capabilities } = req.body;
  agentRegistry.set(name, {
    name, url, tags, capabilities,
    registeredAt: Date.now(),
    lastHeartbeat: Date.now(),
    status: 'alive',
    metrics: null,
  });
  connectAgentEvents(name, url);
  res.json({ ok: true });
});

// Agent 心跳
app.post('/api/agents/heartbeat', (req, res) => {
  const agent = agentRegistry.get(req.body.name);
  if (agent) {
    agent.lastHeartbeat = Date.now();
    agent.status = 'alive';
  }
  res.json({ ok: true });
});

// 定时采集指标（每30秒）
setInterval(async () => {
  for (const [name, agent] of agentRegistry) {
    try {
      const res = await fetch(`${agent.url}/api/metrics`, {
        signal: AbortSignal.timeout(5000),
      });
      agent.metrics = await res.json();
      agent.status = 'alive';
    } catch {
      agent.status = 'dead';
    }
  }
}, 30000);

// 实时事件转发
function connectAgentEvents(name: string, agentUrl: string) {
  const ws = new WebSocket(`${agentUrl.replace('http', 'ws')}/ws`);
  ws.on('message', (data) => {
    const event = JSON.parse(data.toString());
    frontendClients.forEach(client => {
      client.send(JSON.stringify({ agent: name, ...event }));
    });
  });
}
```

> `AgentNode` 数据模型见 [01-核心设计.md](01-核心设计.md) 第 1.7 节。

## 7.7 Dashboard 后端核心职责总结

| 职责 | 说明 |
|---|---|
| 注册发现 | 接收 Agent 注册/心跳/注销，维护在线 Agent 列表 |
| 指标采集 | 定期轮询各 Agent 的 `/api/metrics`，汇总统计数据 |
| 健康检测 | 心跳超时自动标记为离线，恢复后自动标记为在线 |
| 事件转发 | 接收 Agent 实时事件（WebSocket），转发给面板前端 |
| API 网关 | 面板前端通过 Dashboard 后端间接调用 Agent（可选代理模式） |
