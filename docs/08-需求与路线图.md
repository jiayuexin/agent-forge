## 8.2 需求规格

### P0 — 必须实现（MVP）

#### R1: Agent 统一接口

- 所有 Agent 实现 `IAgent` 接口：`init` / `execute` / `stream` / `destroy` / `use` / `on` / `off`
- 类型定义发布为 `@agentforge/types`，零运行时依赖
- Agent 状态机：7 种状态（uninitialized → initializing → ready → running → error → paused → destroyed）

**验收标准：**
- [ ] `IAgent` 类型定义完整且通过 TypeScript 编译
- [ ] 生成的 Agent 代码全部符合 `IAgent` 接口
- [ ] `@agentforge/types` 可独立安装并使用

#### R2: Agent 生成器

- 输入：岗位描述文本（≥ 10 字）
- 输出：完整的 Agent 项目（src / tests / package.json / tsconfig.json / README.md）
- 生成流程：描述解析 → 模板匹配 → Prompt 生成 → 工具推荐 → 代码生成 → 配置生成 → 文档生成 → 编译验证

**验收标准：**
- [ ] 输入 "一个客服Agent，处理用户咨询和投诉" 生成可运行项目
- [ ] 生成的项目 `npm install && npm run build` 无错误
- [ ] 生成的 Agent 可通过 `init` + `execute` 正常执行
- [ ] 生成速度 < 5 秒（不含 LLM 调用）

#### R3: 批量生成

- 支持 JSON / YAML 配置文件批量定义 Agent
- CLI 命令 `agentforge batch agents.yaml`
- 面板批量创建（P1）

**验收标准：**
- [ ] 包含 5 个岗位描述的配置文件，一次性生成 5 个 Agent
- [ ] 每个 Agent 独立完整，可分别使用

#### R4: 三种集成模式

| 模式 | 集成方式 | 目标用户 |
|---|---|---|
| npm 包 | `npm install agent-xxx` → import 使用 | Node.js 开发者 |
| HTTP 服务 | `npx agent serve --port 3001` → REST + SSE | 非 Node.js 项目 |
| SDK 编排 | `@agentforge/sdk` → Pipeline / EventBus / Direct | 系统架构师 |

**验收标准：**
- [ ] npm 模式：< 10 行代码完成 init + execute
- [ ] HTTP 模式：`curl -X POST /api/execute` 返回正确结果
- [ ] SDK 模式：同一进程加载 ≥ 3 个 Agent

#### R5: LLM Provider 抽象

- 3 个内置 Provider：OpenAI / Anthropic / Ollama
- 支持自定义 Provider（任意兼容 OpenAI API 格式的服务）
- 支持代理 baseUrl（适用于国内网络环境）
- 模型注册表（ModelRegistry）：编排器集中管理多个模型端点，Agent 只需引用模型名自动路由
- 同一模型名可注册在不同端点（如 `gpt-4o` 同时在官方 API 和代理），Agent 可按需选择

**验收标准：**
- [ ] OpenAI Provider 调用成功
- [ ] Ollama Provider 连接本地模型成功
- [ ] 自定义 Provider 可通过配置注册
- [ ] 模型注册表配置多个端点，Pipeline 中 Agent 写模型名自动路由到正确端点

#### R6: CLI 工具

| 命令 | 说明 |
|---|---|
| `agentforge create "描述"` | 单个生成 |
| `agentforge batch agents.yaml` | 批量生成 |
| `agentforge serve --port 3001` | 启动 HTTP 服务 |
| `agentforge list` | 列出已生成 Agent |
| `agentforge dashboard` | 启动 Web 面板 |

**验收标准：**
- [ ] 5 个命令均可正常执行
- [ ] `--help` 输出完整用法说明

#### R7: Web 管理面板（基础版）

- Agent 列表页：查看、搜索、状态展示
- 创建 Agent 页：表单输入描述 → 预览 Prompt → 生成代码
- Agent 调试台：三栏布局（会话面板 / 对话区域 / 调试面板）

**验收标准：**
- [ ] 面板可启动访问
- [ ] 列表页展示所有 Agent
- [ ] 调试台可实时对话、流式输出

## 8.3 实施路线图

### Phase 1：核心框架（地基）

**目标**：搭建 Agent 接口标准、运行时核心、基础 Provider

| 任务 | 说明 | 预估 |
|---|---|---|
| `@agentforge/types` | IAgent / AgentConfig / AgentTask / AgentResult 等类型 | 1 天 |
| `BaseAgent` 抽象类 | 生命周期管理、事件系统、插件加载 | 2 天 |
| `AgentRegistry` | Agent 注册与发现 | 1 天 |
| `AgentExecutor` | 任务执行引擎 | 2 天 |
| `OpenAIProvider` | OpenAI API 对接 | 1 天 |
| `AnthropicProvider` | Anthropic API 对接（含 Function Call 适配） | 1 天 |
| `OllamaProvider` | 本地模型对接 | 1 天 |
| 单元测试 | 核心模块覆盖率 ≥ 80% | 2 天 |

### Phase 2：生成引擎（核心能力）

**目标**：从描述到 Agent 代码的自动生成

| 任务 | 说明 | 预估 |
|---|---|---|
| `PromptBuilder` | 岗位描述 → 系统提示词 | 2 天 |
| `TemplateEngine` | EJS 模板渲染 | 1 天 |
| `SkillMatcher` | 根据描述匹配工具集 | 1 天 |
| `CodeEmitter` | 渲染并输出完整项目 | 2 天 |
| 6 套预置模板 | 通用 + 5 个岗位模板 | 3 天 |
| `AgentGenerator` | 串联组件，统一 API | 1 天 |

### Phase 3：CLI + 集成（可用性）

**目标**：命令行工具 + 三种集成模式 + Pipeline 编排

| 任务 | 说明 | 预估 |
|---|---|---|
| `@agentforge/cli` | 5 个核心命令 | 3 天 |
| npm 包模式 | 生成的 Agent 可直接安装引用 | 2 天 |
| HTTP 服务模式 | Express + REST API | 2 天 |
| `@agentforge/sdk` | 框架类 + Pipeline 编排 | 3 天 |
| Pipeline 回退跳转 | 回退信号、拦截器、快照恢复 | 3 天 |
| 集成测试 | 覆盖 3 种集成模式 | 2 天 |

### Phase 4：可视化面板（体验）

**目标**：Web 管理面板 + Agent 调试台

### Phase 5：增强功能（打磨）

## 8.4 成功指标

### 技术指标

| 指标 | 目标 | 衡量方式 | 评估时间 |
|---|---|---|---|
| 生成速度 | 单个 Agent < 5 秒 | CLI 计时 | 上线后 1 周 |
| 生成准确率 | 描述匹配正确率 ≥ 90% | 人工评估 50 个样本 | 上线后 1 个月 |
| 接口兼容性 | 所有生成 Agent 通过接口测试 | 自动化测试 | 每个 Phase |
| 集成便捷度 | npm 集成代码 < 10 行 | 代码行数统计 | 上线后 1 周 |

### 性能指标

| 指标 | 目标 | 衡量方式 | 评估时间 |
|---|---|---|---|
| 冷启动时间 | Agent init < 2 秒 | 性能测试 | Phase 3 |
| 首次响应延迟 | execute 首字 < 3 秒 | 性能测试 | Phase 3 |
| 批量生成上限 | 单次 ≥ 20 个 Agent | 压力测试 | Phase 3 |
| HTTP 并发 | ≥ 100 并发请求 | 压力测试 | Phase 3 |

### 质量指标

| 指标 | 目标 | 衡量方式 |
|---|---|---|
| 核心测试覆盖率 | ≥ 80% | Vitest 报告 |
| 生成代码编译通过率 | 100% | 自动验证 |
| 生成 Agent 核心依赖 | ≤ 2 个 | 依赖检查 |
