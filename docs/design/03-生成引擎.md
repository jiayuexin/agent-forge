## 3.2 Prompt 生成策略

```markdown

{结构化输出定义}

## 限制条件
- {限制1}
- {限制2}
...
```

## 3.3 生成引擎类型定义

```typescript
/** 生成引擎输入 */
interface GenerateInput {
  /** 岗位描述文本（≥ 10 字） */
  description: string;
  /** 显式指定 ClientAgent 名称 */
  name?: string;
  /** 显式指定模板 ID */
  templateId?: string;
  /** 显式指定模型名 */
  model?: string;
  /** 输出目录 */
  outputDir?: string;
  /** ClientAgent 基础配置（由 CLI/SDK 层组装） */
  config?: Partial<ClientAgentConfig>;
  /** 额外自定义数据 */
  custom?: Record<string, unknown>;
}

/** 解析后的岗位描述 */
interface ParsedDescription {
  /** 推导出的角色名称 */
  role: string;
  /** URL-safe 的 Agent 名称 */
  name: string;
  /** 岗位中文标题 */
  displayName: string;
  /** 核心能力标签 */
  capabilities: string[];
  /** 适用场景 */
  scenarios: string[];
  /** 推荐工具类别 */
  toolCategories: string[];
  /** 风险等级 */
  riskLevel: 'low' | 'medium' | 'high';
}

/** 单个模板定义 */
interface TemplateSet {
  /** 模板 ID */
  id: string;
  /** 模板元数据 */
  meta: AgentTemplate;
  /** EJS 模板文件集合；具体生成哪些具名文件（如 main.ts / agent.ts / prompts.ts / tools.ts / runtime.ts）由模板约定 */
  files: Record<string, string>;
}

/** 渲染模板所需数据 */
interface TemplateData {
  /** Agent 身份信息 */
  identity: AgentIdentity;
  /** 解析后的描述 */
  parsed: ParsedDescription;
  /** 系统提示词 */
  systemPrompt: string;
  /** 工具定义列表 */
  tools: ToolDefinition[];
  /** 模板配置 */
  config: Partial<ClientAgentConfig>;
  /** 安全配置 */
  security: ClientAgentSecurityConfig;
  /** 依赖版本 */
  versions: {
    core: string;
    runtimeClient: string;
  };
}

/** 代码输出器上下文 — 由 AgentGenerator 组装后传给 CodeEmitter */
interface EmitContext {
  /** 匹配的模板 */
  template: TemplateSet;
  /** 解析后的描述 */
  parsed: ParsedDescription;
  /** 构建好的系统提示词 */
  systemPrompt: string;
  /** 推荐的工具定义 */
  tools: ToolDefinition[];
  /** 用户输入的生成配置 */
  config: Partial<ClientAgentConfig>;
}

/** 生成结果 */
interface GenerateResult {
  /** 渲染后的文件内容映射；键为相对路径，值为文件内容 */
  files: Record<string, string>;
  /** 解析后的描述元数据 */
  metadata: ParsedDescription;
}

/** 工具推荐器 */
interface SkillMatcher {
  /** 根据解析后的描述与匹配模板推荐工具定义 */
  match(parsed: ParsedDescription, template: TemplateSet): ToolDefinition[];
  /** 注册自定义工具模板 */
  registerTemplate(category: string, tools: ToolDefinition[]): void;
}

/** 代码输出器 */
interface CodeEmitter {
  /** 根据上下文渲染并返回文件内容映射 */
  emit(ctx: EmitContext): Promise<Record<string, string>>;
  /** 验证目标目录是否已存在 */
  exists(outputPath: string): boolean;
}
```

## 3.4 预置模板库

| 模板 ID | 名称 | 适用场景 | 默认风险等级 |
|---|---|---|---|
| `customer-service` | 客服专员 | 用户咨询、投诉处理 | medium |
| `sales-assistant` | 销售顾问 | 产品推荐、报价生成 | medium |
| `code-reviewer` | 代码审查员 | 代码质量审查 | low |
| `content-writer` | 内容创作者 | 文案撰写、翻译 | low |
| `data-analyst` | 数据分析师 | 数据查询、报表分析 | medium |
| `dev-assistant` | 开发助手 | 本地命令执行、Git、文件操作 | high |
| `general` | 通用 Agent | 未匹配其他模板时 | low |

## 3.5 生成物目录结构

生成的 Agent 是一个**可本地运行的客户端应用**，同时包含业务 Agent 代码和客户端运行时：

```
./client-agents/<name>/
├── src/
│   ├── main.ts             # 守护进程入口
│   ├── agent.ts            # Agent 类实现（IClientAgent）
│   ├── prompts.ts          # 系统 Prompt 模板
│   ├── tools.ts            # 工具定义（生成时预置）
│   ├── types.ts            # 本地类型定义
│   └── runtime.ts          # AgentRuntimeClient（连接 Capability Hub）
├── package.json
├── tsconfig.json
├── README.md
└── .agentforge/
    ├── config.json         # Agent 元数据 + Capability Hub 端点配置
    └── security.json       # 本地命令执行授权、敏感操作策略
```

### 关键文件说明

| 文件 | 说明 |
|---|---|
| `src/main.ts` | 守护进程入口，负责启动 Agent、连接 Capability Hub、保持心跳 |
| `src/agent.ts` | ClientAgent 类实现，继承 BaseAgent 并实现 IClientAgent |
| `src/runtime.ts` | 轻量级运行时客户端，注册为 Agent Node，监听远程任务和能力下发 |
| `.agentforge/config.json` | Agent 元数据、Capability Hub 端点、节点显示名称 |
| `.agentforge/security.json` | 本地命令执行授权级别、敏感操作策略、能力安装白名单 |

### runtime.ts 核心职责

`AgentRuntimeClient` 完整类型定义见 [01-核心设计.md §1.15](./01-核心设计.md#115-远程控制与客户端运行时类型)。`runtime.ts` 中的关键启动与消息处理逻辑如下：

```typescript
async function startRuntime(client: IAgentRuntimeClient, agent: IClientAgent): Promise<void> {
  // 1. 调用 agent.init()（若 Agent 尚未初始化）
  // 2. 加载本地能力缓存
  // 3. 连接 Capability Hub WebSocket
  // 4. 注册节点信息
  // 5. 启动心跳
  // 6. 监听 ControlMessage（任务、配置更新、能力下发）
  await client.start();
}

async function onControlMessage(
  client: IAgentRuntimeClient,
  agent: IClientAgent,
  message: ControlMessage,
): Promise<void> {
  if (message.type === 'execute' && 'task' in message.payload) {
    const result = await agent.execute((message.payload as RemoteTask).task);
    client.send({ type: 'result', messageId: message.messageId, nodeId: client.node.id, timestamp: Date.now(), payload: result });
  }
  if (message.type === 'capability-distribute' && 'capability' in message.payload) {
    const ack = await installCapability(message.payload as CapabilityDistributePayload);
    client.send({ type: 'capability-ack', messageId: message.messageId, nodeId: client.node.id, timestamp: Date.now(), payload: ack });
  }
}
```

## 3.6 生成后安全边界

生成引擎输出的是一个**具备本地执行能力的客户端应用**，必须明确其执行边界，防止 LLM 生成结果被直接滥用。

### 默认行为

- 生成产物**不自动执行**，仅写入输出目录。
- 生成完成后返回 `GenerateResult`，包含渲染后的文件内容映射（`files`）与解析后的描述元数据（`metadata`）。
- **本地命令执行默认禁用**，无论模板风险等级如何，用户必须显式授权。

### 自动运行前提

仅当同时满足以下条件时，才允许通过 `--run` 等显式选项直接启动生成的 ClientAgent：

1. TypeScript 编译检查通过。
2. 用户已阅读 `.agentforge/security.json` 并显式确认。
3. 涉及 `high` 风险等级的模板时，必须额外通过 `--confirm-high-risk` 确认。

### 人工 review 要求

包含以下任一因素的生成结果，建议经人工 review 后再启动：

- 模板风险等级为 `high`（如 `dev-assistant`）。
- 预置工具中包含本地命令执行能力。
- 生成过程中工具推荐命中了不在模板预置列表中的工具。

### 禁止行为

生成引擎及 CLI 不得：

- 在未经确认的情况下自动安装依赖或执行 shell 命令。
- 将用户环境变量或 API Key 写入生成文件（`.agentforge/config.json` 仅允许保存 Hub 端点等非敏感配置）。
- 默认启用本地命令执行或远程控制能力。
