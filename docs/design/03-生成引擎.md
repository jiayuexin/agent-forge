## 3.3 预置模板库

| 模板 ID | 名称 | 适用场景 | 默认风险等级 |
|---|---|---|---|
| `customer-service` | 客服专员 | 用户咨询、投诉处理 | medium |
| `sales-assistant` | 销售顾问 | 产品推荐、报价生成 | medium |
| `code-reviewer` | 代码审查员 | 代码质量审查 | low |
| `content-writer` | 内容创作者 | 文案撰写、翻译 | low |
| `data-analyst` | 数据分析师 | 数据查询、报表分析 | medium |
| `dev-assistant` | 开发助手 | 本地命令执行、Git、文件操作 | high |
| `general` | 通用 Agent | 未匹配其他模板时 | low |

## 3.4 生成物目录结构

生成的 Agent 是一个**可本地运行的客户端应用**，同时包含业务 Agent 代码和客户端运行时：

```
./client-agents/<name>/
├── src/
│   ├── main.ts             # 守护进程入口
│   ├── agent.ts            # Agent 类实现（IClientAgent）
│   ├── prompts.ts          # 系统 Prompt 模板
│   ├── tools.ts            # 工具定义（生成时预置）
│   ├── types.ts            # 本地类型定义
│   └── runtime.ts          # AgentRuntimeClient（连接 Capability Hub）
├── package.json
├── tsconfig.json
├── README.md
└── .agentforge/
    ├── config.json         # Agent 元数据 + Capability Hub 端点配置
    └── security.json       # 本地命令执行授权、敏感操作策略
```

### 关键文件说明

| 文件 | 说明 |
|---|---|
| `src/main.ts` | 守护进程入口，负责启动 Agent、连接 Capability Hub、保持心跳 |
| `src/agent.ts` | ClientAgent 类实现，继承 BaseAgent 并实现 IClientAgent |
| `src/runtime.ts` | 轻量级运行时客户端，注册为 Agent Node，监听远程任务和能力下发 |
| `.agentforge/config.json` | Agent 元数据、Capability Hub 端点、节点显示名称 |
| `.agentforge/security.json` | 本地命令执行授权级别、敏感操作策略、能力安装白名单 |

### runtime.ts 核心职责

```typescript
export class AgentRuntimeClient implements IAgentRuntimeClient {
  async start(): Promise<void> {
    // 1. 初始化 Agent
    // 2. 加载本地能力缓存
    // 3. 连接 Capability Hub WebSocket
    // 4. 注册节点信息
    // 5. 启动心跳
    // 6. 监听 ControlMessage（任务、配置更新、能力下发）
  }

  async onControlMessage(message: ControlMessage): Promise<void> {
    if (message.type === 'execute') {
      const result = await this.agent.execute(message.payload.task);
      this.send({ type: 'result', messageId: message.messageId, nodeId: this.nodeId, payload: result });
    }
    if (message.type === 'capability-distribute') {
      const ack = await this.installCapability(message.payload);
      this.send({ type: 'capability-ack', messageId: message.messageId, nodeId: this.nodeId, payload: ack });
    }
  }
}
```

## 3.5 生成后安全边界

生成引擎输出的是一个**具备本地执行能力的客户端应用**，必须明确其执行边界，防止 LLM 生成结果被直接滥用。

### 默认行为

- 生成产物**不自动执行**，仅写入输出目录。
- 生成完成后返回 `GenerateResult`，包含文件列表、元数据和风险等级。
- **本地命令执行默认禁用**，无论模板风险等级如何，用户必须显式授权。

### 自动运行前提

仅当同时满足以下条件时，才允许通过 `--run` 等显式选项直接启动生成的 ClientAgent：

1. TypeScript 编译检查通过。
2. 用户已阅读 `.agentforge/security.json` 并显式确认。
3. 涉及 `high` 风险等级的模板时，必须额外通过 `--confirm-high-risk` 确认。

### 人工 review 要求

包含以下任一因素的生成结果，建议经人工 review 后再启动：

- 模板风险等级为 `high`（如 `dev-assistant`）。
- 预置工具中包含本地命令执行能力。
- 生成过程中工具推荐命中了不在模板预置列表中的工具。

### 禁止行为

生成引擎及 CLI 不得：

- 在未经确认的情况下自动安装依赖或执行 shell 命令。
- 将用户环境变量或 API Key 写入生成文件（`.agentforge/config.json` 仅允许保存 Hub 端点等非敏感配置）。
- 默认启用本地命令执行或远程控制能力。
