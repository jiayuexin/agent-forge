## 6.3 核心页面说明

**首页/概览 `/`**

- 在线节点数量、能力数量、ClientAgent 模板数量
- 近期远程执行任务统计
- 快速创建 ClientAgent 入口

**ClientAgent 模板列表 `/client-agents`**

- 卡片式展示所有已生成的 ClientAgent 模板
- 显示：名称、岗位、关联节点数、最后调用时间
- 操作：配置、重新生成、测试、删除

**创建 ClientAgent `/client-agents/create`**

- 表单输入：名称、描述、模型选择、模板选择
- 实时预览生成的 Prompt
- 创建成功后跳转到详情页

**节点列表 `/nodes`**

- 展示所有已连接的 ClientAgent 节点
- 显示：节点名称、IP、状态、运行的 Agent、最后心跳
- 操作：查看详情、远程触发、断开连接

**节点详情 `/nodes/:id`**

- 节点基本信息和主机信息
- 已安装能力列表
- 远程发送任务并查看结果
- 实时日志和事件流
- 本地命令执行授权状态

**节点对话 `/nodes/:id/chat`**

- 与节点上的 Agent 实时对话
- 支持流式输出
- 显示调用链路

**能力管理 `/capabilities`**

- 列表展示所有 Tool / Skill / Plugin
- 显示：名称、类型、风险等级、版本、下发状态
- 操作：编辑、发布新版本、下发、撤销

**能力市场 `/capabilities/market`**

- 浏览团队内部可用的能力
- 查看能力文档、版本历史、风险等级
- 一键下发到指定节点或节点组

## 6.5 Capability Hub 控制中心

Capability Hub 不仅是监控面板，更是**能力中心 + 控制中心**。它维护所有 ClientAgent 节点的注册表，通过 WebSocket 下发控制命令和能力，并实时展示节点状态和事件。

### 架构总览

```
┌─────────────────────────────────────────────────────────────────┐
│                     Capability Hub                               │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────────┐  │
│  │  React 前端   │  │  Hub 后端     │  │   编排器（可选）      │  │
│  │  :8080       │  │  :8080       │  │                      │  │
│  └──────┬───────┘  └──────┬───────┘  └──────────┬───────────┘  │
│         │                 │                      │              │
│         └─────────────────┼──────────────────────┘              │
│                           │  HTTP / WebSocket                   │
└───────────────────────────┼─────────────────────────────────────┘
                            │
         ┌──────────────────┼──────────────────┐
         │                  │                  │
┌────────▼────────┐ ┌───────▼────────┐ ┌──────▼────────┐
│  ClientAgent 1   │ │  ClientAgent 2  │ │  ClientAgent 3 │
│ AgentRuntimeClient│ │ AgentRuntimeClient│ │ AgentRuntimeClient│
│ + IClientAgent    │ │ + IClientAgent    │ │ + IClientAgent    │
└─────────────────┘ └─────────────────┘ └─────────────────┘
```

### 通信链路

```
AgentRuntimeClient ──注册/心跳/状态/指标───► Capability Hub
AgentRuntimeClient ◄──控制命令────────────── Capability Hub
AgentRuntimeClient ──实时事件──────────────► Capability Hub ──► 面板前端
```

## 6.6 客户端运行时：启动时注册 + 保持连接

```typescript
import { AgentRuntimeClient } from '@agentforge/runtime-client';

const runtime = new AgentRuntimeClient({
  hubUrl: 'wss://hub.example.com',
  authToken: process.env.AGENTFORGE_NODE_TOKEN,
  nodeName: '开发机-北京办公室',
  tags: ['dev-assistant', 'beijing'],
});

await runtime.start();

// 启动后自动执行：
// 1. 通过 WebSocket 向 Hub 发送注册消息
// 2. 每 30 秒发送心跳
// 3. 监听 ControlMessage（execute / stream / config-update / capability-distribute / ping）
// 4. 上报状态、指标、事件
// 5. 关闭时发送注销消息
```

## 6.7 远程触发示例

Capability Hub 向客户端节点下发任务：

```typescript
const controlMessage: ControlMessage = {
  type: 'execute',
  messageId: 'msg-001',
  nodeId: 'node-dev-assistant-001',
  timestamp: Date.now(),
  payload: {
    taskId: 'task-001',
    type: 'chat',
    task: {
      type: 'chat',
      input: { message: '当前仓库状态如何？' },
    },
    source: 'hub',
  },
};

websocket.to(nodeId).send(JSON.stringify(controlMessage));
```

客户端执行后返回：

```typescript
const agentMessage: AgentMessage = {
  type: 'result',
  messageId: 'msg-001',
  nodeId: 'node-dev-assistant-001',
  timestamp: Date.now(),
  payload: {
    success: true,
    output: { content: '当前仓库没有未提交的改动。' },
    meta: { duration: 1200, model: 'gpt-4o' },
  },
};
```

## 6.8 Capability Hub 后端核心职责

| 职责 | 说明 |
|---|---|
| 节点注册发现 | 接收 AgentRuntimeClient 注册/心跳/注销，维护在线节点列表 |
| 命令下发 | 通过 WebSocket 向指定节点发送控制命令 |
| 能力管理 | 维护 Tool / Skill / Plugin 定义和版本 |
| 能力下发 | 向指定 ClientAgent 推送能力，接收 capability-ack |
| 状态/指标采集 | 接收节点主动上报的状态和指标 |
| 健康检测 | 心跳超时自动标记节点离线 |
| 事件转发 | 接收节点实时事件，转发给面板前端 |
| 认证授权 | 校验客户端 token，控制命令和能力下发权限 |

## 6.9 远程管控信任模型

Capability Hub 远程控制 ClientAgent 涉及跨网络命令下发，必须建立清晰的认证、鉴权与隔离边界。

### 认证

- `AgentRuntimeClient` 连接 `/ws/nodes/:nodeId` 时，必须通过 query 参数或首部携带 `authToken`。
- Hub 后端校验 token 的签名、有效期与作用域，校验失败立即断开连接。

### 节点隔离

- 普通节点 token 只能操作自身对应的 `nodeId`。
- Hub 管理员 token 可查看全部节点，但敏感命令仍需按白名单校验。
- 不同 token 的节点注册表在逻辑上相互隔离。

### 命令白名单

| 类型 | 说明 | 是否允许远程下发 |
|---|---|---|
| `execute` | 执行单次任务 | ✅ |
| `stream` | 流式执行任务 | ✅ |
| `config-update` | 更新运行时配置 | ✅（仅允许非敏感字段） |
| `capability-distribute` | 下发能力 | ✅（需签名校验） |
| `ping` | 心跳探测 | ✅ |
| `stop` | 停止当前任务/断开连接 | ✅ |
| `code-load` | 远程加载代码 | ❌ |
| `dependency-install` | 远程安装依赖 | ❌ |
| `shell-exec` | 远程执行 shell | ❌ |

### 敏感操作本地确认

涉及 `sensitiveOperations`（如本地命令执行、资金、签约、权限变更、数据删除）的任务，即使来自 Hub，也必须在 ClientAgent 本地由运行者确认，或预先在 `.agentforge/security.json` 中显式授权。
