## 6.3 核心页面说明

**首页/概览 `/`**

- 在线节点数量、Agent 模板数量
- 近期远程执行任务统计
- 快速创建入口

**Agent 列表 `/agents`**

- 卡片式展示所有 Agent 模板
- 显示：名称、岗位、关联节点数、最近调用时间
- 操作：配置、生成代码、测试、删除

**创建 Agent `/agents/create`**

- 表单输入：名称、描述、模型选择、工具勾选
- 实时预览生成的 Prompt
- 创建成功后跳转到详情页

**节点列表 `/nodes`**（新增）

- 展示所有已连接的客户端 Agent 节点
- 显示：节点名称、IP、状态、运行的 Agent、最后心跳
- 操作：查看详情、远程触发、断开连接

**节点详情 `/nodes/:id`**（新增）

- 节点基本信息和主机信息
- 能力列表
- 远程发送任务并查看结果
- 实时日志和事件流

**节点对话 `/nodes/:id/chat`**（新增）

- 与节点上的 Agent 实时对话
- 支持流式输出
- 显示调用链路

## 6.4 Agent 调试台 `/playground`

生成 Agent 后的核心验证环境，支持交互式对话测试和深度调试。

```
│   会话面板    │       对话区域            │      调试面板              │
│              │                          │                           │
│  📋 对话历史   │  👤 用户: 帮我审查这段...    │  📊 本次调用统计            │
│  ├─ 会话 1    │                          │  ├─ 响应耗时: 2.3s        │
│  ├─ 会话 2    │  🤖 Agent: 我发现以下...  │  ├─ Token: 1,245         │

│              │                          │  └─ 成本: $0.008         │
│  ⚙ 调试配置   │                          │                           │
│  ├─ 模型选择   │  ┌──────────────────┐   │  🔍 调用链路               │
│  ├─ 温度      │  │  输入消息...       │   │  ├─ System Prompt 构建 ✅  │

│  └─ 🔌 插件市场 │  [发送] [清空] [重试]      │  ├─ 二次 LLM 调用 ✅      │
│              │                          │  └─ 结构化输出解析 ✅      │
└──────────────┴──────────────────────────┴───────────────────────────┘
```

**核心功能：**

| 功能 | 说明 |
|---|---|
| Agent 选择 | 顶部下拉选择要测试的 Agent，支持按名称/标签搜索 |
| 实时对话 | 流式输出（SSE），支持 Markdown 渲染、代码高亮、LaTeX 数学公式 |
| 调试配置 | 可实时切换模型（OpenAI / Anthropic / Ollama）、调整温度、MaxTokens、开关特定工具，无需重启 Agent |
| 工具插拔 | 调试时动态挂载/卸载工具插件，测试 Agent 在不同工具组合下的表现 |
| 临时 Mock 工具 | 不接入真实后端，用自定义 Mock 数据模拟工具返回，测试 Agent 的工具调用逻辑 |
| 调用链路追踪 | 展示每次对话的完整调用链：System Prompt 构建 → LLM API 调用 → 工具调用（如有） → 二次 LLM → 结构化输出解析 |
| 工具调用详情 | 点击调用链路中的工具节点，查看入参、出参、耗时 |
| Token 统计 | 实时显示 Prompt Tokens / Completion Tokens / Total Tokens，估算成本 |
| 会话管理 | 保存多组对话会话，可切换、清空、导出为 JSON |
| 变量注入 | 支持设置上下文变量（如 `{orderId}`），对话中可引用，模拟真实业务场景 |
| 对比测试 | 同一输入，同时发给不同模型或不同配置的 Agent，左右对比输出效果 |
| 导出报告 | 将对话记录 + 调用链路 + 统计数据导出为 Markdown / JSON 报告，便于分享和存档 |

# 7. Dashboard 控制中心

> Dashboard 不仅是监控面板，更是**云端控制中心**。它维护所有客户端 Agent 节点的注册表，通过 WebSocket 下发控制命令，并实时展示节点状态和事件。

## 7.2 客户端运行时：启动时注册 + 保持连接

```typescript
import { AgentRuntimeClient } from 'agent-customer-service/runtime';

const runtime = new AgentRuntimeClient({
  dashboardUrl: 'wss://dashboard.example.com',
  authToken: process.env.AGENTFORGE_NODE_TOKEN,
  nodeName: '客服节点-北京办公室',
  tags: ['customer-service', 'beijing'],
});

await runtime.start();

// 启动后自动执行：
// 1. 通过 WebSocket 向 Dashboard 发送注册消息
// 2. 每 30 秒发送心跳
// 3. 监听 ControlMessage（execute / stream / config-update / ping）
// 4. 上报状态、指标、事件
// 5. 关闭时发送注销消息
```

## 7.4 Dashboard 后端：节点注册表 + 命令转发

```typescript
const nodeRegistry = new Map<string, AgentNode>();
const nodeSockets = new Map<string, WebSocket>();

// WebSocket 连接建立时
wss.on('connection', (ws, req) => {
  const nodeId = extractNodeId(req);
  nodeSockets.set(nodeId, ws);

  ws.on('message', (data) => {
    const message: AgentMessage = JSON.parse(data.toString());

    switch (message.type) {
      case 'status':
        updateNodeStatus(message.nodeId, message.payload);
        break;
      case 'metric':
        updateNodeMetrics(message.nodeId, message.payload);
        break;
      case 'event':
        broadcastToFrontend(message);
        break;
      case 'result':
      case 'stream-chunk':
        forwardToRequester(message);
        break;
    }
  });

  ws.on('close', () => {
    nodeSockets.delete(nodeId);
    markNodeOffline(nodeId);
  });
});

// 向指定节点发送控制命令
function sendControl(nodeId: string, message: ControlMessage): boolean {
  const ws = nodeSockets.get(nodeId);
  if (!ws || ws.readyState !== WebSocket.OPEN) return false;
  ws.send(JSON.stringify(message));
  return true;
}
```

> `AgentNode`、`ControlMessage`、`AgentMessage` 数据模型见 [01-核心设计.md](./01-核心设计.md) 第 1.7 节和第 1.13 节。

## 7.7 Dashboard 后端核心职责总结

| 职责 | 说明 |
|---|---|
| 节点注册发现 | 接收 Agent Runtime Client 注册/心跳/注销，维护在线节点列表 |
| 命令下发 | 通过 WebSocket 向指定节点发送 `execute`/`stream`/`config-update` 等控制命令 |
| 状态/指标采集 | 接收节点主动上报的状态和指标 |
| 健康检测 | 心跳超时自动标记节点离线 |
| 事件转发 | 接收节点实时事件，转发给面板前端 |
| 认证授权 | 校验客户端 token，控制命令权限校验 |

## 7.8 远程管控信任模型

Dashboard 远程控制客户端 Agent 涉及跨网络命令下发，必须建立清晰的认证、鉴权与隔离边界。

### 认证

- `AgentRuntimeClient` 连接 `/ws/nodes/:nodeId` 时，必须通过 query 参数或首部携带 `authToken`。
- Dashboard 后端校验 token 的签名、有效期与作用域，校验失败立即断开连接。

### 节点隔离

- 普通节点 token 只能操作自身对应的 `nodeId`，不能查看或控制其他节点。
- Dashboard 管理员 token 可查看全部节点，但敏感命令仍需按白名单校验。
- 不同 token 的节点注册表在逻辑上相互隔离。

### 命令白名单

WebSocket 控制消息类型限定为以下范围：

| 类型 | 说明 | 是否允许远程下发 |
|---|---|---|
| `execute` | 执行单次任务 | ✅ |
| `stream` | 流式执行任务 | ✅ |
| `config-update` | 更新运行时配置 | ✅（仅允许非敏感字段） |
| `ping` | 心跳探测 | ✅ |
| `stop` | 停止当前任务/断开连接 | ✅ |
| `code-load` | 远程加载代码 | ❌ |
| `dependency-install` | 远程安装依赖 | ❌ |
| `shell-exec` | 远程执行 shell | ❌ |

### 敏感操作本地确认

涉及 `sensitiveOperations`（如退款、签约、权限变更、数据删除）的任务，即使来自 Dashboard，也必须在客户端本地由运行者确认，或预先在 `.agentforge.json` 中显式授权。

### 不引入的能力

v1 不引入 RBAC、多租户、审计日志持久化等超出当前范围的模型。
