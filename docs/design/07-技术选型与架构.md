## 命名规范

- 工具名: kebab-case (`query-order`, `create-refund`)
- ClientAgent 目录名: `<name>` (默认 `client-agent-<role>`)
- StatelessAgent 类名: `<Role>Agent`
- 能力 ID: `tool-<name>`, `skill-<name>`, `plugin-<name>`
- CLI 命令: `agentforge <command>`
- API 路径: `/api/` 前缀

## 7.5 生成的 ClientAgent 项目结构

```
client-agents/<name>/
├── src/
│   ├── main.ts                  # 守护进程入口
│   ├── agent.ts                 # ClientAgent 核心实现
│   ├── prompts.ts               # 系统提示词定义
│   ├── tools.ts                 # 预置工具定义
│   ├── types.ts                 # 类型定义
│   └── runtime.ts               # AgentRuntimeClient 配置
├── package.json
├── tsconfig.json
├── README.md
└── .agentforge/
    ├── config.json              # Agent 元数据 + Hub 端点
    └── security.json            # 本地命令授权 + 敏感策略
```

## 7.6 架构设计原则

| 原则 | 说明 |
|---|---|
| 接口优先 | 所有 Agent 遵循 `IAgent` 统一接口，ClientAgent 与 StatelessAgent 可互换调用 |
| 插件化 | 核心功能通过 Middleware 和 Plugin 机制扩展 |
| 能力可分发 | Tool/Skill 下发定义，Plugin 下发签名脚本，支持离线缓存 |
| 约定优于配置 | 合理默认值，80% 场景零配置即可使用 |
| 可观测性 | 内置日志、指标、追踪，方便监控和调试 |
| 安全优先 | 本地命令执行默认禁用，敏感操作必须本地确认 |
