# 10. 安全模型

> **文档层级**: 第二层 · 设计规格
> **文档类型**: 设计规格
> **文档状态**: 已定稿
> **文档版本**: docs-v0.4
> **最后更新**: 2026-06-23
> **实现状态**: 已完成
>
> AgentForge 的安全设计总览，涵盖本地命令执行授权、能力下发安全、敏感操作确认、认证鉴权与沙箱隔离。

---

## 10.1 安全设计原则

1. **默认最小权限**：ClientAgent 默认禁止本地命令执行，默认禁止远程敏感操作。
2. **能力下发受控**：Tool/Skill 下发定义，Plugin 下发签名脚本，禁止无限制远程代码执行。
3. **敏感操作本地确认**：涉及资金、权限变更、数据删除、本地命令执行等操作，必须获得本地用户确认。
4. **不引入兜底逻辑**：遵循"正常业务流程优先"原则，不主动添加降级、容错、重试、熔断等机制。

---

## 10.2 本地命令执行授权

ClientAgent 具备调用本地终端/PowerShell 执行命令的能力，这是系统最高风险能力之一。

### 授权级别

`LocalCommandAuthLevel` 与 `LocalCommandAuthConfig` 类型定义见 [01-核心设计.md §1.2](./01-核心设计.md#12-clientagent-与-statelessagent-扩展接口)。本节仅保留行为说明，不重复声明类型。

| 级别 | 说明 | 适用场景 |
|---|---|---|
| `disabled` | 默认状态，禁止执行任何命令 | 所有新生成 Agent |
| `readonly` | 只允许只读命令（如 `ls`、`ps`、`git status`） | 普通查询型 Agent |
| `whitelist` | 只允许白名单内的命令 | 需要特定命令的 Agent |
| `full` | 开放命令执行，敏感命令仍需二次确认 | 受信任的开发环境 |

### 默认只读命令白名单

```typescript
const DEFAULT_READONLY_COMMANDS = [
  'ls', 'pwd', 'cat', 'echo', 'ps', 'top', 'df', 'du',
  'git status', 'git log', 'git diff', 'git branch',
  'Get-Process', 'Get-Location', 'Get-ChildItem',
];
```

### 敏感命令二次确认

以下命令即使授权级别为 `full`，执行前也必须本地弹窗/命令行确认：

```typescript
const DEFAULT_SENSITIVE_COMMANDS = [
  'rm', 'rmdir', 'del', 'format', 'mkfs',
  'git push', 'git reset --hard', 'git clean -fd',
  'sudo', 'su', 'chmod', 'chown',
];
```

## 10.3 能力下发安全

### 下发内容分层

| 能力类型 | 下发内容 | 安全要求 |
|---|---|---|
| Tool | JSON Schema + 端点声明 | Hub 管理员发布 |
| Skill | Tool 列表 + Prompt 模板 | Hub 管理员发布 |
| Plugin | 签名脚本包 | 必须签名校验 + 沙箱运行 |

### 签名校验

Plugin 能力下发时必须携带签名，完整结构定义见 [09-能力市场与下发.md §9.4](./09-能力市场与下发.md#94-plugin-定义)。ClientAgent 安装 Plugin 前：

1. 从本地信任库获取对应公钥。
2. 验证签名是否有效。
3. 验证签名是否过期。
4. 验证 Plugin 是否在安装白名单内。

签名无效或过期的 Plugin 拒绝安装。

### 沙箱运行

Plugin 运行在受限沙箱中：

```typescript
interface SandboxConfig {
  /** 超时(ms)，默认 30000 */
  timeout?: number;
  /** 内存限制(MB)，默认 128 */
  memoryLimit?: number;
  /** 网络访问权限 */
  network?: 'none' | 'hub-only' | 'full';
  /** 文件系统访问权限 */
  filesystem?: 'none' | 'readonly' | 'restricted' | 'full';
}
```

> 能力白名单的权威来源是 `PluginCapability.allowedCapabilities`（定义见 [09-能力市场与下发.md §9.4](./09-能力市场与下发.md#94-plugin-定义)）。沙箱实例启动时，运行时会从对应 `PluginCapability.allowedCapabilities` 读取并复制到沙箱的只读白名单，确保运行时与声明一致；`SandboxConfig` 本身不声明该字段。

默认沙箱配置：

- `network: 'none'`
- `filesystem: 'readonly'`
- 能力白名单默认为空（运行时从 `PluginCapability.allowedCapabilities` 读取；未声明时按空数组处理）

## 10.4 敏感操作本地确认

### 本地确认辅助函数

```typescript
/** 判断任务是否涉及敏感操作 */
declare function isSensitiveTask(task: AgentTask): boolean;

/** 向本地用户展示确认对话框，返回用户是否同意 */
declare function askLocalUserConfirmation(task: AgentTask): Promise<boolean>;

/** 上报本地确认请求并等待用户响应 */
declare function waitForLocalApproval(
  requestId: string,
  options?: { timeout?: number },
): Promise<boolean>;
```

### 触发本地确认的场景

- 任务涉及 `sensitiveOperations`（如 `refund`、`contract-sign`、`payment`、`data-delete`）。
- 任务需要执行 `requireConfirmationFor` 中的本地命令。
- 安装/更新 Plugin 能力。
- 修改本地安全配置。

### 确认流程

```typescript
// ClientAgent 收到 execute 消息
if (message.type === 'execute' && message.payload && 'task' in message.payload) {
  const task = (message.payload as RemoteTask).task;
  if (isSensitiveTask(task)) {
    // 1. 先弹窗/命令行询问本地用户
    const localConfirmed = await askLocalUserConfirmation(task);
    if (!localConfirmed) {
      this.send({ type: 'error', messageId: message.messageId, nodeId: this.node.id, timestamp: Date.now(), payload: { code: 'USER_REJECTED', message: '用户拒绝执行' } });
      return;
    }

    // 2. 向 Capability Hub 上报 local-approval-request，等待 Hub 侧最终确认
    const request: LocalApprovalRequest = {
      requestId: generateId(),
      type: 'sensitive-operation',
      description: '该任务涉及敏感操作，是否继续？',
      summary: { task },
    };
    this.send({ type: 'local-approval-request', nodeId: this.node.id, timestamp: Date.now(), payload: request });

    const hubConfirmed = await waitForLocalApproval(request.requestId, { timeout: 60000 });
    if (!hubConfirmed) {
      this.send({ type: 'error', messageId: message.messageId, nodeId: this.node.id, timestamp: Date.now(), payload: { code: 'USER_REJECTED', message: 'Hub 拒绝执行' } });
      return;
    }
  }
}
```

## 10.5 Capability Hub 认证鉴权

### Token 类型

| Token 类型 | 权限 | 适用场景 |
|---|---|---|
| 节点 Token | 只能操作自身 nodeId | ClientAgent 连接 Hub |
| 管理员 Token | 可管理全部节点和能力 | Hub 管理操作 |
| 只读 Token | 只能查看节点状态和能力 | 监控/审计 |

### Token 内容

```typescript
interface HubToken {
  /** Token 类型 */
  type: 'node' | 'admin' | 'readonly';
  /** 主题 */
  subject: string;
  /** 可访问的 nodeId（node token） */
  nodeId?: string;
  /** 过期时间 */
  exp: number;
  /** 签发时间 */
  iat: number;
}
```

### 鉴权规则

- 节点 Token 只能发送/接收自身 nodeId 的消息。
- 管理员 Token 可查看全部节点，但下发敏感能力仍需额外审批。
- 只读 Token 不能发送控制命令。

## 10.6 数据安全

### API Key 管理

- API Key 仅通过环境变量注入。
- 禁止将 API Key 写入 `.agentforge/config.json` 或生成文件。
- Capability Hub 的 `/api/config` 端点自动脱敏：`apiKey` 字段替换为 `***`。

### 本地缓存安全

- 能力缓存目录 `.agentforge/capabilities/` 默认仅当前用户可读写。
- Plugin 脚本包安装前验证签名。
- 缓存的 Capability 定义不可被外部程序随意修改。

### 执行日志

- 所有本地命令执行记录到审计日志。
- 审计日志包含：命令内容、执行时间、执行结果、触发来源。
- 审计日志可配置上报到 Capability Hub。

---

## 10.7 AI 安全

### 提示注入防护

- Tool 描述中明确参数类型和取值范围。
- 结构化输出使用 JSON Schema 校验。
- 不信任的用户输入不直接拼接到系统 Prompt 中。

### 工具滥用防护

- 每个 Capability（Tool / Skill / Plugin / Agent）声明 `riskLevel` 和 `sensitiveOperations`；`ToolDefinition` 本身不包含这两个字段。
- 高风险工具调用需本地确认。
- 工具调用次数上限由 `FrameworkConfig.maxToolCalls` 控制。

### 输出校验

- AgentResult 的输出需符合声明的 `outputSchema`。
- 结构化输出失败时返回错误，不 fallback 到纯文本。
