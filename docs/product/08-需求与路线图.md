## 8.2 需求规格

### P0 — 必须实现

#### R1: Agent 统一接口

- 所有 Agent 实现 `IAgent` 接口：`init` / `execute` / `stream` / `destroy` / `use` / `on` / `off`
- 支持两种形态：`ClientAgent`（客户端应用）和 `StatelessAgent`（无状态编排单元）
- 类型定义发布为 `@agentforge/types`，零运行时依赖
- Agent 状态机：7 种状态（含 ClientAgent 的 `daemon-running` 状态）

**验收标准：**
- [ ] `IAgent`、`IClientAgent`、`IStatelessAgent` 类型定义完整
- [ ] 生成的 ClientAgent 符合 `IClientAgent` 接口
- [ ] `@agentforge/types` 可独立安装并使用

#### R2: ClientAgent 生成器

- 输入：岗位描述文本（≥ 10 字）
- 输出：完整的 ClientAgent 客户端应用（src / .agentforge / package.json / tsconfig.json / README.md）
- 生成流程：描述解析 → 模板匹配 → Prompt 生成 → 工具推荐 → 代码渲染 → 项目配置 → 安全配置 → 文档生成 → 编译验证

**验收标准：**
- [ ] 输入 "一个能执行 Git 命令的本地编程助手" 生成可运行项目
- [ ] 生成的项目 `npm install && npm run build` 无错误
- [ ] 生成的 ClientAgent 可通过 `agentforge run` 正常启动
- [ ] 生成速度 < 5 秒（不含 LLM 调用）

#### R3: 批量生成

- 支持 JSON / YAML 配置文件批量定义 ClientAgent
- CLI 命令 `agentforge batch client-agents.yaml`
- 面板批量创建（P1）

**验收标准：**
- [ ] 包含 5 个岗位描述的配置文件，一次性生成 5 个 ClientAgent
- [ ] 每个 ClientAgent 独立完整，可分别运行

#### R4: ClientAgent 运行与 Capability Hub 连接

- `agentforge run` 启动本地守护进程
- ClientAgent 通过 WebSocket 连接 Capability Hub
- 支持注册、心跳、状态上报、远程任务执行、能力下发

**验收标准：**
- [ ] ClientAgent 启动后成功注册到 Hub
- [ ] Hub 可向 ClientAgent 下发 execute 任务并返回结果
- [ ] 断网后 ClientAgent 仍可执行本地已缓存能力

#### R5: LLM Provider 抽象与 ModelRegistry

- 3 个内置 Provider：OpenAI / Anthropic / Ollama
- 支持自定义 Provider（任意兼容 OpenAI API 格式的服务）
- 支持代理 baseUrl
- `ModelRegistry`：编排器集中管理多个模型端点；Agent 步骤只引用模型名即可自动路由
- 解析规则：步骤显式 endpoint > 第一个注册该模型的端点 > Pipeline 默认模型 + 默认端点 > 抛出 `ModelNotFoundError`

**验收标准：**
- [ ] OpenAI Provider 调用成功
- [ ] Ollama Provider 连接本地模型成功
- [ ] 自定义 Provider 可通过配置注册
- [ ] `AgentFramework` 初始化时可注册多个模型端点
- [ ] 同一模型名（如 `gpt-4o`）存在于多个端点时，按解析规则正确路由
- [ ] 未匹配到任何端点时抛出 `ModelNotFoundError`

#### R6: CLI 工具

| 命令 | 说明 |
|---|---|
| `agentforge create "描述"` | 生成 ClientAgent |
| `agentforge batch config.yaml` | 批量生成 ClientAgent |
| `agentforge run [path]` | 启动 ClientAgent 守护进程 |
| `agentforge list` | 列出已生成的 ClientAgent |
| `agentforge dashboard` | 启动 Capability Hub |
| `agentforge serve [path]` | 启动本地调试 HTTP 服务 |
| `agentforge capability` | 能力市场管理 |

**验收标准：**
- [ ] 7 个命令均可正常执行
- [ ] `--help` 输出完整用法说明

#### R7: Capability Hub 基础版

- ClientAgent 模板列表页
- 创建 ClientAgent 页：表单输入描述 → 预览 Prompt → 生成代码
- 节点列表页：查看、搜索、状态展示
- Agent 调试台：三栏布局（会话面板 / 对话区域 / 调试面板）

**验收标准：**
- [ ] Hub 可启动访问
- [ ] 列表页展示所有 ClientAgent 模板和节点
- [ ] 调试台可实时对话、流式输出

#### R8: 安全模型（本地命令、能力下发、认证鉴权、审计）

- 默认禁用本地命令执行，支持 `disabled` / `readonly` / `whitelist` / `full` 四级授权
- 敏感本地命令执行前需用户二次确认
- Capability Hub 向 ClientAgent 下发的 Plugin 必须经管理员签名，ClientAgent 安装前校验签名
- ClientAgent 连接 Capability Hub 必须携带节点认证 Token；节点 Token 只能操作自身 `nodeId`
- 本地命令执行、能力下发、安全配置变更均需记录审计日志

**验收标准：**
- [ ] 默认状态下 ClientAgent 拒绝执行本地命令
- [ ] 用户授权后可执行对应级别命令
- [ ] `rm`、`git push` 等敏感命令触发本地确认
- [ ] 签名无效的 Plugin 被 ClientAgent 拒绝安装并上报失败
- [ ] 未携带有效 Token 的 ClientAgent 无法注册到 Hub
- [ ] 节点 Token 无法操作其他节点
- [ ] 审计日志可查询最近 90 天的本地命令、能力下发、配置变更记录

---

#### R9: StatelessAgent 编排（基础 Pipeline）

- SDK 支持注册多个 StatelessAgent
- 支持顺序执行 `.add()`、并行执行 `.parallel()`、条件分支 `.branch()`、分叉 `.fork()`
- 支持 Pipeline 级默认模型与步骤级模型引用
- 支持步骤间数据传递

**验收标准：**
- [ ] 3 个 StatelessAgent 顺序流水线成功执行
- [ ] 2 个 StatelessAgent 并行步骤成功执行
- [ ] 步骤可引用 `ModelRegistry` 中注册的模型
- [ ] StatelessAgent 可通过 Capability Hub 调用 ClientAgent 的远程能力

## 8.3 实施路线图

### Phase 1：核心运行时

**目标**：搭建 Agent 接口标准、BaseAgent、Provider 适配

| 任务 | 说明 | 预估 |
|---|---|---|
| `@agentforge/types` | IAgent / ClientAgent / StatelessAgent / Capability 类型 | 1 天 |
| `BaseAgent` 抽象类 | 生命周期管理、事件系统、插件加载 | 2 天 |
| `ClientAgent` 抽象类 | 守护进程、Hub 连接、能力缓存 | 3 天 |
| `StatelessAgent` 抽象类 | 进程内无状态执行 | 1 天 |
| `OpenAIProvider` / `AnthropicProvider` / `OllamaProvider` | Provider 适配 | 3 天 |
| 单元测试 | 核心模块覆盖率 ≥ 80% | 2 天 |

### Phase 2：生成引擎

**目标**：从描述到 ClientAgent 客户端应用的自动生成

| 任务 | 说明 | 预估 |
|---|---|---|
| `PromptBuilder` | 岗位描述 → 系统提示词 | 2 天 |
| `TemplateEngine` | EJS 模板渲染客户端应用 | 2 天 |
| `CodeEmitter` | 渲染并输出完整项目 | 2 天 |
| 7 套预置模板 | 通用 + 6 个岗位模板 | 3 天 |
| `AgentGenerator` | 串联组件，统一 API | 1 天 |

### Phase 3：CLI + Capability Hub

**目标**：命令行工具 + ClientAgent 运行 + 能力中心

| 任务 | 说明 | 预估 |
|---|---|---|
| `@agentforge/cli` | create / run / dashboard / capability 命令 | 4 天 |
| `@agentforge/runtime-client` | WebSocket 连接、心跳、能力缓存 | 4 天 |
| Capability Hub 后端 | 节点注册、能力管理、下发协议 | 4 天 |
| Capability Hub 前端 | 节点列表、能力市场、调试台 | 5 天 |
| 集成测试 | ClientAgent 运行 + Hub 集成 | 3 天 |

### Phase 4：SDK 编排

**目标**：StatelessAgent 编排与 ClientAgent 协同

| 任务 | 说明 | 预估 |
|---|---|---|
| `@agentforge/sdk` | AgentFramework + CapabilityRegistry | 3 天 |

| ClientAgent 远程能力调用 | 编排中调用 ClientAgent 能力 | 3 天 |

## 8.4 成功指标

### 技术指标

| 指标 | 目标 | 衡量方式 | 评估时间 |
|---|---|---|---|
| 生成速度 | 单个 ClientAgent < 5 秒 | CLI 计时 | 上线后 1 周 |
| 生成准确率 | 描述匹配正确率 ≥ 90% | 人工评估 50 个样本 | 上线后 1 个月 |
| 接口兼容性 | 所有生成 ClientAgent 通过接口测试 | 自动化测试 | 每个 Phase |
| 能力下发成功率 | ≥ 99% | 自动化测试 | Phase 3 |

### 性能指标

| 指标 | 目标 | 衡量方式 | 评估时间 |
|---|---|---|---|
| ClientAgent 启动时间 | < 3 秒 | 性能测试 | Phase 3 |
| 首次响应延迟 | execute 首字 < 3 秒 | 性能测试 | Phase 3 |
| 批量生成上限 | 单次 ≥ 10 个 ClientAgent | 压力测试 | Phase 3 |
| Hub WebSocket 并发 | ≥ 100 节点 | 压力测试 | Phase 3 |

### 质量指标

| 指标 | 目标 | 衡量方式 |
|---|---|---|
| 核心测试覆盖率 | ≥ 80% | Vitest 报告 |
| 生成代码编译通过率 | 100% | 自动验证 |
| 生成 ClientAgent 核心依赖 | ≤ 2 个 | 依赖检查 |
